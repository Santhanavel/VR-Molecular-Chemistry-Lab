1. Purpose & Scope
This document serves as the primary reference for building the VR Molecular Chemistry Lab â€” a fully interactive Meta Quest application where students physically grab atom tokens in VR and combine them to form valid molecules. It covers the high-level goals, system architecture, Unity project layout, and the relationship between all five design documents in this series.
1.1  What We Are Building
A standalone VR experience that runs on Android (Meta Quest 2/3/Pro) in which:
Players stand inside a virtual chemistry lab environment.
Four atom types â€” Hydrogen (H), Oxygen (O), Carbon (C), Nitrogen (N) â€” exist as grabbable 3D tokens.
Bringing atoms close together triggers a real-time bond validation system.
Valid combinations instantly spawn a 3D molecule prefab with labelled bonds.
A world-space Library Panel tracks every discovered molecule.
A Molecule Inspector Panel shows detailed bond information when the player examines a molecule.
Sound effects and haptic feedback reward every successful bond.
1.2  What We Are NOT Building
A flat-screen / non-VR application.
A simulation with physics-accurate electron orbitals or quantum mechanics.
Multiplayer or cloud-synced progress.
Molecules beyond the 18 defined in the assessment spec.

3. System Architecture â€” Three Layers
The application is structured in three clearly separated layers. This separation keeps each concern testable in isolation and prevents monolithic scripts.
3.1  Layer 1 â€” XR Input
Provided entirely by Unity packages. No custom code lives here; we only subscribe to events.

OpenXR Plugin
Translates raw Quest controller / hand-tracking data into Unity XR input actions

Enables hand-tracking via MediaPipe; atom grab works with pinch gesture
3.2  Layer 2 â€” Logic / Game Systems (Our Code)
Five C# classes handle all application logic. They communicate through direct method calls and C# events â€” never via FindObjectOfType at runtime.
